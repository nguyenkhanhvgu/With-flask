Database Concept: Categories

Categories demonstrate a One-to-Many relationship where:

  • One category can have many posts
  • Each post belongs to zero or one category (optional relationship)
  • The foreign key category_id in the Post table references the Category table
  • Categories can exist without posts, but posts reference existing categories

Blueprint Organization:

This page demonstrates how blueprints organize related functionality:

  • All blog-related routes are grouped under the blog blueprint
  • URL patterns are consistent: /blog/categories, /blog/category/<id>
  • Templates are organized in the templates/blog/ directory
  • Related functionality is kept together for better maintainability
